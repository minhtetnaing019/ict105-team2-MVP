# Problem Notes

## Selected Semester Project Opportunity
### Project Title
Peer Tutoring Booking Platform

## Target Users
- Undergraduate students who need academic help in specific subjects.
- Students who are willing to tutor others in subjects they are strong in.

## Problem Statement
Students who struggle with a subject often do not know who among their peers is willing or qualified to tutor them. At the same time, students who are capable of tutoring have no easy way to make themselves known. This leads to wasted time, unmet academic needs, and missed opportunities for peer-to-peer learning.

## Current Alternatives
Students currently rely on:
- Word-of-mouth recommendations
- Asking classmates directly
- Posting in course or year-group chat groups

### Limitations
- Reaches only a limited audience
- No structured scheduling system
- Difficult to confirm tutor availability
- Difficult to verify subject expertise

## Proposed IT Solution
A web-based platform where:
- Tutors can create profiles
- Tutors can list subjects they can teach
- Tutors can set available time slots
- Students can browse tutor listings
- Students can book tutoring sessions directly with tutors

## Why This Opportunity Was Selected
The Peer Tutoring Booking Platform received the highest overall score during the team's NUF evaluation.

Reasons:
- High usefulness for students
- Feasible to develop within one semester
- Reasonable novelty since no dedicated peer tutoring platform currently exists on campus
- More controllable and testable than alternatives such as a canteen status board or ride-share application
- Does not rely on real-time data from third parties

## Feasibility
This project can be developed without IoT or advanced cybersecurity requirements because:

- No physical sensors or hardware are needed
- No real-time device integration is required
- User data is relatively low risk
- No financial or highly sensitive personal information is stored
- Standard authentication and web security practices are sufficient
- Can be implemented using common web development technologies

## Possible MVP Type
HTML / CSS / JavaScript Prototype

### MVP Features
#### Student Features
- Register/Login
- Search tutors by subject
- View tutor profiles
- Book tutoring sessions
- View booking history

#### Tutor Features
- Register/Login
- Create tutor profile
- List subjects offered
- Set available time slots
- Manage bookings

## Expected Benefits
- Easier access to academic support
- Improved peer-to-peer learning
- Better visibility for tutors
- Organized scheduling and booking process
- Reduced time spent finding academic help
