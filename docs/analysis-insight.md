# Lab 08 – Analytics Insights

## Overview
This analysis evaluates the usability testing results of the Peer Tutoring Booking Platform's booking workflow.

## Key Metrics

| Metric | Result |
|--------|--------|
| Total Testers | 5 |
| Completed Tasks | 4 |
| Task Success Rate | 80% |
| Average Feedback Score | 3.6 / 5 |
| Average Interest Level | 4.2 / 5 |

## Key Findings

- 80% of testers successfully completed the booking workflow.
- Most users showed strong interest in using the platform.
- The main usability issue was the unclear booking status labels.
- Some users were confused by the time-slot and notes fields.
- The tutor search bar needs better visibility.

## Interpretation

The results validate the core booking workflow of the MVP. Most users were able to find a tutor and submit a booking request successfully. The remaining issues are minor usability problems that can be resolved through clearer UI labels and improved form design.

## Next Improvements

- Add a status legend or tooltip for booking status.
- Replace the free-text time-slot field with a structured time-slot picker.
- Add placeholder text for the notes field.
- Increase the visibility of the tutor search bar.

## Conclusion

The MVP successfully demonstrates the core functionality of the Peer Tutoring Booking Platform. Future iterations will focus on improving usability through small interface enhancements rather than redesigning the workflow.
