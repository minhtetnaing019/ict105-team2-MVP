## Objective
Define and justify the technical architecture for the Peer Tutoring Booking Platform MVP.

## Proposed Architecture
- Architecture Style: Three-tier web application
- Frontend: HTML, CSS, JavaScript
- Backend: Node.js with Express.js
- Database: MySQL
- Version Control: GitHub
- Development Tool: Visual Studio Code

## Components
### Frontend
- User interface for students and tutors
- Login and registration pages
- Tutor listing page
- Booking form
- User dashboard

### Backend
- Authentication
- User management
- Tutor management
- Booking management
- REST API for frontend communication

### Database
Main tables:
- Users
- Tutors
- Subjects
- Bookings

## Justification
This architecture is suitable because:
- Easy to develop and maintain.
- Separates presentation, business logic, and data.
- Supports future expansion.
- Works well for an MVP prototype.
