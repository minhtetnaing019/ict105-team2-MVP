# Privacy and Data Protection

## Project Title
**Peer Tutoring Booking Platform**

## Purpose
This document explains what data is collected by the prototype, why it is needed, and how the team protects user privacy.

## Data Collection

| Data Field | Purpose | Personal Data | Sensitive Data | Decision |
|------------|---------|---------------|----------------|----------|
| Student Name | Identify booking owner | Yes | No | Keep |
| Tutor Name | Display tutor information | Yes | No | Keep |
| Subject | Match tutor and student | No | No | Keep |
| Time Slot | Schedule tutoring session | No | No | Keep |
| Booking Notes | Give tutor additional information | Possibly | No | Keep with caution |
| Email | Not required for MVP | Yes | No | Remove |
| Phone Number | Not required for MVP | Yes | No | Remove |

## Privacy Rules

- Collect only the minimum information required.
- Do not collect unnecessary personal information.
- Do not collect payment information.
- Do not collect government identification numbers.
- Students should not enter sensitive personal information in the Notes field.

## Data Visibility

| User | Accessible Data |
|------|-----------------|
| Student | Own booking only |
| Tutor | Own profile |
| Admin | All booking records |

## Data Protection

- Prototype uses sample/demo data only.
- Personal information is not shared publicly.
- Student booking information is visible only to the owner and admin.
- Tutor profiles are public because tutors voluntarily join the platform.

## Conclusion

The prototype follows data minimization principles by collecting only information required to demonstrate the booking process.
