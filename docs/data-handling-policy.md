# Data Handling Policy

## Project Title
**Peer Tutoring Booking Platform**

## Purpose

This document explains how data is collected, stored, accessed, and protected during prototype development.

## Data Collection

The prototype collects:

- Student Name
- Subject
- Tutor Selection
- Booking Time
- Optional Booking Notes

The prototype does NOT collect:

- Email Address
- Phone Number
- Payment Information
- Government ID

## Data Storage

Prototype data is stored using:

- JSON files
- localStorage
- Sample CSV files

No real database is used.

## Data Access

| Role | Access |
|------|--------|
| Student | Own bookings |
| Tutor | Own profile |
| Admin | All records |

## Data Protection Rules

- Use fictional sample data.
- Avoid unnecessary personal information.
- Do not publish private student records.
- Admin functions should be separated from student pages.

## Data Retention

Prototype data is temporary and used only for demonstration.

## Responsible Use

All team members agree to use data only for ICT105 coursework.
