# Wireframe Specification

## Project
**Peer Tutoring Booking Platform**

### Purpose
The wireframes describe the main user interface screens required for the MVP. They show how students and tutors interact with the system before implementation.

---

# Screen Descriptions

## 1. Homepage
### Purpose
Provide an entry point for students and tutors.

### Main Components
- Project logo
- Navigation menu
- Login button
- Register button
- Search Tutor button
- Become a Tutor button
- About section
- Footer

---

## 2. Booking Form

### Purpose
Allow students to request a tutoring session.

### Main Components
- Student Name
- Subject
- Tutor Selection
- Date Picker
- Time Picker
- Notes
- Submit Button
- Confirmation/Error Message

---

## 3. Tutor List

### Purpose
Allow students to browse available tutors.

### Main Components
- Search box
- Subject filter
- Tutor cards
- Tutor name
- Subject
- Availability
- Rating
- View Details button

---

## 4. Tutor Detail

### Purpose
Display complete tutor information.

### Main Components
- Tutor photo
- Tutor name
- Subjects taught
- Biography
- Available schedule
- Contact information
- Book Now button

---

## 5. Dashboard

### Purpose
Provide users with a summary of their tutoring activities.

### Main Components
- Total bookings
- Upcoming sessions
- Completed sessions
- Statistics cards
- Activity summary

---

## 6. Admin View

### Purpose
Allow administrators to manage users and bookings.

### Main Components
- User management table
- Booking management
- Tutor approval
- Edit button
- Delete button
- Update Status button
- Search bar

---

# Wireframe Quality Rules

- Use consistent navigation across all screens.
- Use meaningful labels and buttons.
- Display realistic sample data.
- Ensure every screen supports at least one functional requirement.
- Maintain a simple and user-friendly layout.
